# Bug Report: Bus Path Resolution Bug

## Issue Found During Test Run

### Symptom

When running `adhd.bus.resolve()` from inside the `adhd/` submodule directory, the bus path resolves to:

```
~/.brainxio/adhd/adhd/bus.jsonl
```

This is **double-nested** — the repo slug is "adhd" and the package directory is also "adhd", producing a redundant path.

### Expected Behavior

The bus should resolve to the **parent project** slug (`.projects` in this case), not the submodule directory name. When ADHD is used as a submodule inside a larger workspace, the bus should represent the workspace root, not the individual package.

### Root Cause

`resolve()` calls `git rev-parse --show-toplevel` which returns the submodule root (`adhd/`), and then takes `Path(toplevel).name` which is `"adhd"`.

When ADHD is a submodule, the git toplevel is the submodule directory, not the parent repo.

### Proposed Fix

1. **Environment variable override** — Already exists (`ADHD_BUS_REPO_SLUG`), but it's not documented in the README's env var table.

2. **Parent repo detection** — When inside a submodule, `resolve()` should detect the parent repo. This can be done via:
   ```bash
   git rev-parse --show-superproject-working-tree
   ```
   (Git 2.13+) which returns the parent repo root, or empty if not in a submodule.

3. **Fallback chain**:
   - `ADHD_BUS_REPO_SLUG` env var
   - `git rev-parse --show-superproject-working-tree` (parent repo)
   - `git rev-parse --show-toplevel` (current repo, could be submodule)
   - `"default"`

### Test Evidence

```python
import adhd.bus as bus
print(bus.resolve())
# Actual:   PosixPath('/home/mrrobot0985/.brainxio/adhd/adhd/bus.jsonl')
# Expected: PosixPath('/home/mrrobot0985/.brainxio/adhd/projects/bus.jsonl')
```
